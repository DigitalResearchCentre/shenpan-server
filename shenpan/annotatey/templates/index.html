Welcome to Annotation.Bham

Annotation.Bham is a microservice for making and documenting textual decisions.

You can access the service through the web, however, if you want to help develop the service, or run it on your own computer, you can check out Shenpan.

Textual editors make many decisions, such as when regularizing collations or enhancing document searches. This service aims to record those decisions, so that textual editing can be repeatable and verifiable.

It is currenty in closed pre-alpha.

Use cases

We are especially targeting to two initial use cases, one related to collation of textual witnesses, another related to search in documents.

  • In the first use case, editors indicate and identify relationships between variants, and the tool generates rules from that for a regularization tool rule engine, or gives the added information back to another service, such as CollateX.
  • In the second use case, scholars query documents and qualify individual hits with respect to their relevance to the search they had in mind. The tool generates rules from their decisions which can then be applied to a new, more focused or smarter search. This use case may be applied to search for lexical items, rhymes, clausulae, or even themes.

Basic Usage

To make a new decision requires do calling the service using a POST request and providing JSON input to http://www.annotation.bham.ac.uk/api.

For any single regularization or qualification based on a user decision, the following items can be stored as a new rule:

  • Token: The text for which the decision applies
  • Type: The type of decision: regularize or not; equivalent or not.
  • Context: where the user was when the decision was made
  • Scope: how far the decision applies: local, document, or global.
  • Lemma (optional): The base form for regulariation decision
  • External username (optional): The user that made the decision.
  • Description (optional): Freeform justification/description.
